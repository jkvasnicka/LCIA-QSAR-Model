Further Comparison of Workflows: LCIA QSAR Project¶

Date: April 20, 2023

This notebook builds on the comparison between our original draft workflow with that of Nicolo Aurisano.

First, I refactored our workflow in several ways:

  • Replaced the original metals filter with a less strict filter that excludes chemicals only if they lack a "QSAR-ready" SMILES
  • Updated the features to include the experimental LD50s (median) instead of the TEST predictions.
  • Using a different transformer function to reduce skew during feature preprocessing. In particular, I noticed that my original Log10Transformer can be problematic for skewed features with a small proportion of negative values in the dataset. The algorithm identifies which columns to transform based on the training set to avoid "data leakage." There were some cases in which some negative values were sampled in the test set but not the training set, which led to errors in the log-transformation. As an alternative, this notebook uses the Yeo-Johnson power transform which accepts negative values: https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.PowerTransformer.html#sklearn.preprocessing.PowerTransformer

This notebook compares the following workflows:

  1. Original draft workflow with updates described above
  1. Alternative workflow that uses only chemicals with complete data, while omitting both feature selection and imputation

Compare estimators with repeated internal cross-validation¶

Use 50 randomized repeats of a 5-fold cross-validation. The same data partitions are used for each estimator to enable comparison.

In [7]:
def repeated_kfold_for_estimators(
        estimators, X, y, preprocessing_steps, 
        n_splits=5, n_repeats=50, random_state=1):
    '''Helper function which implements a repeated cross-validation for each estimator.
    '''
    # Inialize a container. 
    cv_results = {}
    for k, est in estimators.items():
        
        pipe = Pipeline(steps=preprocessing_steps + [('estimator', est)])
        
        cv_results[k] = learn.repeated_kfold(
            pipe, 
            X, 
            y, 
            learn.regression_metrics, 
            n_splits=n_splits, 
            n_repeats=n_repeats, 
            random_state=random_state
        )
            
    return pd.concat(cv_results, axis=1)

Permutation feature importances (repeated)¶

Implement the same repeated 5-fold cross-validation with the addition of an internal loop which computes permutation feature importance scores on the test data. This algorithm is relatively expensive, so I restricted the number of permutations to 5 and returned the mean scores across permutations. The description statistics that follow are therefore computed on these means values.

Note of caution¶

I did not use these importance scores for any feature selection because I was worried about optimistically biasing the model ("data leakage") without using a nested cross-validation strategy.

In [16]:
def repeated_permutation_importance(
        estimator, X, y, scoring=None, n_splits=5, n_cv_repeats=50, 
        n_permutation_repeats=5, random_state=1):
    '''Compute permutation feature importances with repeated cross-validation.
    '''
    if scoring is None:
        scoring = 'r2'
    
    if hasattr(X, 'columns'):
        columns = X.columns
    else:
        columns = pd.RangeIndex(X.shape[-1])
        
    X, y = learn.ensure_numpy_ndarrays(X, y)

    rkf = RepeatedKFold(
        n_splits=n_splits, 
        n_repeats=n_cv_repeats, 
        random_state=random_state)

    # Initialize a container.
    permutation_results = []
    for train_ix, test_ix in rkf.split(X):
        X_train, X_test = X[train_ix, :], X[test_ix, :]
        y_train, y_test = y[train_ix], y[test_ix]
        estimator.fit(X_train, y_train)

        results = permutation_importance(
            estimator,
            X_test, 
            y_test,
            scoring=scoring,
            n_repeats=n_permutation_repeats, 
            random_state=random_state, 
            n_jobs=-1)
        permutation_results.append(results['importances_mean'])
    
    return pd.DataFrame(permutation_results, columns=columns)
