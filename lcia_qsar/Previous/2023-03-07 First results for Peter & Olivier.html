Predicting Oral Points of Departure (Preliminary):
LCIA QSAR Project

Author: Jacob Kvasnicka
Date: March 7, 2023

This notebook implements a preliminary Random Forest pipeline to predict oral points of departure for the LCIA QSAR Project for non-reproductive/developmental effects. This analysis consists of the following main sections:

  1. Use the original feature set of 12 physical-chemical property estimates by Nicolò Aurisano.
  2. Use a larger set of 226 features including updated physical-chemical property estimates from CompTox, and molecular descriptors from the RDKit Python library. For the updated property estimates, some implausible predicted values are filtered out (e.g., negative water solubility). Use the median experimental value across all data sources, if present, else use the median predicted value.
  3. Assess the relevance of these new features using a permutation approach.

For the Random Forest, a similar approach is used as Nicolò Aurisano with the following notable differences in this analysis:

  1. Filter out chemicals that are inappropriate for QSAR modeling:
    • Drop any chemicals missing a molecular formula or DTXSID (e.g., minerals, mixtures, etc.)
    • Drop any chemicals that contain metal.

As was done by Nicolò Aurisano, exclude any chemicals with study count <= 3.

  1. Implement a feature preprocessing pipeline:
    1. Drop constant features
    2. Drop features with missing values exceeding threshold % (10% default)
    3. Impute missing values with medians
    4. Remove strong correlations among features (default > 0.9, Pearson)
  2. Implement 1-fold validation (80/20% train/test) while accepting default hyperparameter values. Later, can improve.

def features_preprocessing_pipeline(
        X_train, X_test, missing_thres=0.1, corr_thres=0.9, 
        corr_method=None):
    '''
    '''
    if corr_method is None:
        corr_method = 'pearson'
        
    # Drop constant features.
    const_columns = features.constant_columns(X_train)
    X_train = X_train.drop(const_columns, axis=1)
    X_test = X_test.drop(const_columns, axis=1)

    # Drop features with proportions of missing values > threshold.
    missing_columns = features.columns_missing_exceeding(
        X_train, missing_thres)
    X_train = X_train.drop(missing_columns, axis=1)
    X_test = X_test.drop(missing_columns, axis=1)

    # For all features, impute missing values with the medians.
    medians = X_train.quantile(0.5)
    X_train = X_train.fillna(medians)
    X_test = X_test.fillna(medians)

    # Remove strong correlations among the features.
    corr_columns = features.correlated_columns(
        X_train, corr_thres, corr_method)
    X_train = X_train.drop(corr_columns, axis=1)
    X_test = X_test.drop(corr_columns, axis=1)

    return X_train, X_test

As a first pass, a 1-fold validation will be used. The data are randomly split into separate training (80%) and test (20%) sets. The default hyperparameter values will be accepted for now, which may lead to some overfitting. Later, these hyperparameters could be tuned using a grid-search algorithm.

def random_forest_regression_pipeline(
        X, y, test_size=0.2, random_state=1):
    '''
    '''
    # Get the intersection of chemicals. 
    X, y = features.inner_join_split(X, y)
    
    X_train, X_test, y_train, y_test = train_test_split(
        X, 
        y, 
        test_size=test_size,
        random_state=random_state)

    X_train, X_test = features_preprocessing_pipeline(
        X_train, X_test)

    estimator = RandomForestRegressor(
        n_jobs=-1,
        random_state=random_state)
    estimator.fit(X_train, y_train)
    y_pred = estimator.predict(X_test)
    
    results = {
        'X_train': X_train, 
        'X_test': X_test, 
        'y_train': y_train, 
        'y_test': y_test, 
        'estimator': estimator, 
        'y_pred': y_pred }
        
    return  results

def plot_model_evaluation(
        X_test, y_test, y_pred, estimator, title, **kwargs):
    '''Helper function to plot regression from a results dictionary.
    '''
    score = '$R^2$ = ' + str(estimator.score(X_test, y_test).round(2))
    xlabel = 'Predicted log10(POD [mg/kg-d]'
    ylabel = 'Actual log10(POD [mg/kg-d]'

    fig, ax = plot_regression(
        y_pred, 
        y_test, 
        score, 
        title, 
        xlabel, 
        ylabel)
    
    return fig, ax
    
def plot_regression(
        x, y, score, title=None, xlabel=None, ylabel=None):
    '''
    '''
    fig, ax = plt.subplots()

    ax.scatter(
        x, 
        y, 
        color='black')

    # Plot the one-one line for comparison.
    ax.plot(
        y, 
        y, 
        color='red', 
        linestyle='--', 
        linewidth=0.75)

    bbox = dict(
        boxstyle='round', 
        fc='blanchedalmond', 
        ec='orange', 
        alpha=0.5
    )
    text_position = (0.95, 0.07)
    ax.text(
        *text_position,
        score, 
        fontsize=10, 
        bbox=bbox,
        transform=ax.transAxes, 
        horizontalalignment='right')

    ax.set_title(title)
    ax.set_xlabel(xlabel)
    ax.set_ylabel(ylabel)
    
    return fig, ax

Assess feature importances¶

Permutation feature importance is used to identify any irrelevant features in the new feature set. Permutation feature importance is defined as the mean decrease in a model's performance score (e.g., R2) when a given feature value is randomly shuffled K times. This approach has several advantages over the tree-based measure of feature importance that uses the mean decrease in impurity:

  • Model agnostic and can be calculated many times with different permutations of the feature
  • Unbiased with respect to high cardinality features (i.e., many unique values) vs. low cardinality features
  • Can be computed on a held-out test or validation set, making it possible to highlight which features contribute the most to the generalization power of the model.

The cells below compute permutation feature importances across the new feature set, using R2 as the evaluation metric, and K = 30 permutations.
